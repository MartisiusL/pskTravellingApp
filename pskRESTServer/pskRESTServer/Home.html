PSK Travelling App API

Existing rest calls:

Api/User - returns list of all users
Api/User/501 - returns one user with specified ID (501 - 509)

Api/Trip - returns list of all trips
Api/Trip/401 - returns one trip with specified ID (401 - 404)

Api/Office - returns list of all offices
Api/Office/201 - returns one office with specified ID (201 - 203)

Api/Authorization/test@gmail.com/123456 - authorization request, returns one account ID for given email and password if they are correct, otherwise returns -1 (some emails: test, abc, def, ghi ... ends with @gmail.com; all passwords are: 123456)

There are each call to delete any value by simply calling method Delete with link "api/{insert the word trip/user/account/office}/id", but will not give links for that here.
